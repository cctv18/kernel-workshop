# 安卓内核创意工坊
- 这里是我（[@cctv18](https://github.com/cctv18)）为我的Android内核编译脚本（目前包括[6.1](https://github.com/cctv18/oppo_oplus_realme_sm8650)，[6.6](https://github.com/cctv18/oppo_oplus_realme_sm8750)，[6.12](https://github.com/cctv18/oppo_oplus_realme_sm8850)系列内核）准备的创意工坊仓库。
### 这里有什么内容？
- 安卓内核自定义过程中可开启的各种功能选项介绍，第三方/移植功能代码补丁，内核编译所需的工具（某些脚本/二进制程序等），可用于自定义内核的一切资源。
### 如果我有好的想法或代码，是否可以贡献到这里？
- 当然！创建这个仓库的目的就是希望爱好内核编程的朋友们能够有一个交流共享内核自定义技巧的平台。如果你有好的内核编程技巧、希望开发/测试的功能、自定义补丁或第三方工具等，欢迎fork此仓库，把自己的资源上传到自己fork的仓库后，通过GitHub仓库的Pull Request（简称PR）功能，向我的仓库发起合并申请，把自己想要分享或让大家测试的内容提交到这里，以便大家共享，共同测试，共同开发，共同进步；
- 如果你有某些好的想法，但尚不清楚具体实现方法，可在Isuues区发布Issue，提出你希望引入的内核功能/技术，并让大家讨论。
### 我可以提交什么资源？
- 目前该仓库计划的可提交资源有：
- 内核功能源码/补丁（各厂商开源功能源码、第三方内核代码、移植代码、安全补丁、自研代码等）；
- 内核编译所需的某些小工具（实现某些功能的便利脚本、编译辅助程序、处理二进制文件的某些工具、环境支持等）；
- 内核功能配置方案（需提供开启/关闭/配置的具体选项及实现的功能）；
- 其他内核编译技巧（编译过程中的编译选项配置，环境配置，代码修正/修改指令等）；
- 其他内核编程相关内容或以上资源的组合。
### 我应该如何正确地向这里贡献资源？
- 首先，请把你需要贡献的资源整理为便于他人使用、查看的格式（源码文件、补丁文件、二进制文件、.md格式文档等），不要使用带密码的压缩包或不提供任何功能/使用说明，对于过大的工具/工具链建议直接使用原始链接，通过shell指令拉取（GitHub仓库有文件大小上传限制）；
- 在提交PR前，请你确保你的代码/程序等至少能正常完成一个版本内核的GitHub工作流编译流程（可不限于我的6.1/6.6/6.12内核；至于是否能开机可让他人自行测试，由于不同内核源码、设备环境情况复杂，自定义代码本身就难以保证有良好的可移植性）；
- 对于二进制类程序/文件，请在PR时提供出处/原始代码等，以证明其中不含恶意代码；
- 在提交的PR中，请对你的提交内容进行简要的说明，说明其功能/基本的使用方法，对于复杂的代码/工具/教程等可单独提供一篇Markdown（.md格式）文档进行详细说明；
- 对于尚在测试，不稳定的功能/代码，请在提交PR时标题中加入[WIP]标头（即Working in progress）以提示他人；
- 请确保你提交的资源是公开的、不含恶意代码/广告/其他有害信息的、不涉及他人隐私/企业机密的，否则造成的一切法律后果由提交者本人自负，与本仓库/仓库所有者无关。
### 我应该如何正确地使用这里的资源？
- 在使用内核创意工坊及这里的资源自定义内核前，请确保你对基本的Linux shell指令语法、GitHub工作流运行逻辑及手机救砖方法有一定的了解，使用这里的资源自定义内核导致的手机无法启动等各种异常问题与本仓库所有者/资源提交者无关；
- 在开始使用我的仓库中的自定义脚本前，请把脚本开头的agree_custom值改为1，才能正常运行脚本（防止有人在不了解内核工坊使用方法/误触的情况下开启自定义脚本）；
- 在工作流中引用这里的资源时，请注意引入时自定义脚本的执行路径及你使用的指令，避免执行路径出错或语法问题导致的意外bug；
- 本仓库原则上不对自定义脚本导致的功能异常负责，但如果你确定你编译出的内核出现的异常与该仓库中的某些代码有关，或你经过多次测试可以在某些特定机型/环境中复现某些问题，你可以在本仓库发布Issue，凭报错日志（工作流/系统日志）及稳定复现该问题的条件，联系资源提交者（GitHub的Issue中可通过用户名艾特对应用户）或其他开发者讨论问题的发生原因及解决方法。
